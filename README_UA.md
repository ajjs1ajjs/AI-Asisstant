## 🚀 Встановлення

### Вимоги

- **ОС**: Windows 10/11, Linux, macOS
- **RAM**: мінімум 4GB (рекомендується 8GB+)
- **Python**: 3.9+

### Крок 1: Встановлення залежностей

```bash
pip install -r requirements.txt
```

**Важливо:** `sentence-transformers` може завантажуватися довго (~500MB)

### Крок 2: Запуск

```bash
python main.py
```

Або через batch файл:
```bash
start.bat
```

## 🔀 Git Інтеграція

### Відкрити Git панель

`Git → Git Статус` або `🔀 Git`

### Основні операції

#### Ініціалізація

```
Git → Ініціалізувати
```

Створює новий Git репозиторій в проекті

#### Commit

1. Відкрийте Git панель
2. Вкладка **Status** - перегляд змін
3. Виділіть файли → **Add Selected** або **Add All**
4. Вкладка **Commit** - введіть повідомлення → **Commit**

#### Push/Pull

1. Відкрийте Git панель
2. Вкладка **Гілки**
3. **⬆️ Push** - відправити зміни
4. **⬇️ Pull** - отримати зміни

#### Гілки

- Перегляд існуючих гілок
- Створення нової гілки
- Перемикання між гілками (double-click)

#### Clone

```
Git → Clone
```

Введіть URL репозиторію та оберіть директорію

## 📊 Бенчмарки

### Запуск бенчмарку

`Модель → Бенчмарк моделей`

### Що тестується

1. **Швидкість генерації** - токенів/секунду
2. **Час завантаження** - секунди
3. **Якість** - суб'єктивна оцінка (1-10)

### Результати

Зберігаються в `~/.ai-ide/benchmarks.json`

Можна експортувати в CSV для порівняння

### Рекомендації

| Швидкість | Якість | Рекомендація |
|-----------|--------|--------------|
| >30 tok/s | Будь-яка | ✅ Чудово для кодування |
| 10-30 tok/s | 7+ | ✅ Добре для чату |
| <10 tok/s | Будь-яка | ⚠️ Повільно, спробуйте меншу модель |

## 📚 Контекстний пошук

### Індексація проекту

`Інструменти → Індексувати проект`

AI проіндексує всі файли з розширеннями:
`.py, .md, .txt, .js, .ts, .cpp, .h, .java, .go, .rs...`

### Як це працює

```
Файли → Чанки → Ембединги → FAISS індекс
              ↓
Запит → Пошук схожих чанків → Контекст для AI
```

### Переваги

- ✅ **Семантичний пошук** - розуміє зміст, не тільки ключові слова
- ✅ **Швидкий** - мілісекунди навіть для 1000+ чанків
- ✅ **Точний** - cosine similarity для релевантності

### Очищення контексту

`Редагування → Очистити контекст`

## 🛠️ Вирішення проблем

### "llama-cpp-python не встановлено"

```bash
pip install llama-cpp-python
```

Або з CUDA:
```bash
pip install llama-cpp-python --extra-index-url https://abetlen.github.io/llama-cpp-python/whl/cu121
```

### "sentence-transformers не встановлено"

```bash
pip install sentence-transformers
```

### "Not enough memory"

- Оберіть меншу модель
- Закрийте інші програми
- Зменшіть `n_ctx` в налаштуваннях

### Модель завантажується довго

- Великі моделі (32B) можуть завантажуватися 30+ хв
- Спробуйте дзеркало: `ghproxy.com`
- Перевірте інтернет з'єднання

### FAISS не працює

```bash
pip install faiss-cpu
```

### Git не знайдено

Встановіть Git: https://git-scm.com/download

### Повільна генерація

- Зменшіть `n_ctx` (розмір контексту)
- Збільшіть `temperature` (0.5-0.7)
- Використовуйте меншу модель

## 🎯 Поради

### Для слабких ПК (4-8GB RAM)

1. Використовуйте моделі до 7B
2. Зменшіть контекст до 2048
3. Вимкніть автоіндексацію
4. Індексуйте тільки ключові файли

### Для потужних ПК (16GB+ RAM)

1. Спробуйте 14B-32B моделі
2. Збільшіть контекст до 8192
3. Увімкніть GPU шари (якщо є NVIDIA)
4. Індексуйте весь проект

### Для найкращої якості коду

1. **Qwen2.5-Coder-7B** (8GB RAM) - найкраще співвідношення
2. **Qwen2.5-Coder-14B** (16GB RAM) - відмінна якість
3. **Qwen2.5-Coder-32B** (32GB RAM) - SOTA для коду
